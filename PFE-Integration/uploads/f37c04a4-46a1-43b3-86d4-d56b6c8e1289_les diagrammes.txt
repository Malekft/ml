        Note over Notif, Email: Après 60 secondes...
        Notif->>DB: INSERT Notification (INTERNE)
        Notif->>Email: sendEmailForNotification()

    end

    %% ========================
    %% TRAITEMENT ADMIN
    %% ========================

3)sequence presence en temps reel :

sequenceDiagram
    autonumber
    participant E as Employé
    participant App as Application Angular
    participant WS as Serveur WebSocket (Spring)
    participant DB as Base de données
    participant A as Administrateur

    Note over E, A: Synchronisation de la présence en temps réel

    %% Connexion
    E->>App: Ouverture de l'application
    App->>WS: Connexion (Négociation + Token JWT)
    WS->>DB: MISE À JOUR : utilisateur.est_en_ligne = VRAI
    
    WS-->>App: Abonnement au canal /topic/presence
    WS-->>A: Diffusion : "L'employé X est en ligne"
    
    Note right of A: L'indicateur de présence passe au vert instantanément
    
    %% Déconnexion
    E->>App: Fermeture du navigateur ou Déconnexion
    App->>WS: Demande de déconnexion
    WS->>DB: MISE À JOUR : utilisateur.est_en_ligne = FAUX
    WS-->>A: Diffusion : "L'employé X est hors ligne"
    
    Note right of A: L'indicateur de présence passe au gris

1. La Phase de Connexion (L'ouverture de l'app)
Connexion (Handshake + JWT) : Dès que l'employé ouvre l'application Angular, le PresenceService demande au serveur Spring Boot d'ouvrir un "tunnel" de communication appelé WebSocket. On envoie le jeton JWT pour prouver qui est l'utilisateur.
UPDATE Database : Le serveur reçoit la connexion, identifie l'utilisateur, et va tout de suite mettre à jour sa fiche dans la base de données (est_en_ligne = true).
Subscribe & Broadcast :
L'application de l'employé s'abonne (Subscribe) à un canal pour recevoir des nouvelles.
Le serveur envoie un message flash (Broadcast) à tous les autres utilisateurs (notamment l'Administrateur) pour dire : "Hé, l'employé X vient de se connecter !".
2. L'Effet Visuel (Le Temps Réel)
Le point vert : L'écran de l'administrateur reçoit ce message via le tunnel WebSocket et met à jour l'interface instantanément. C'est ce qui donne l'aspect "Live" à ta plateforme HR.
3. La Phase de Déconnexion (La fermeture de l'app)
Disconnect : Quand l'employé ferme l'onglet de son navigateur ou clique sur "Déconnexion", le tunnel se brise.
UPDATE Database : Le serveur détecte la coupure, et remet le statut à false en base de données.
Broadcast : Le serveur prévient tout le monde : "L'employé X est parti". Le point vert devient gris sur le tableau de bord de l'admin.
Pourquoi c'est important pour ton PFE ?
Technologie STOMP/WebSocket : Cela montre que tu ne fais pas que des requêtes HTTP simples (Request/Response), mais que tu maîtrises la communication bidirectionnelle.
Optimisation : Cela évite que l'administrateur ne doive cliquer sur "Actualiser" toutes les 5 minutes pour voir qui est là.
Expérience Utilisateur (UX) : Ton application ressemble à un outil moderne comme Slack ou Teams.
